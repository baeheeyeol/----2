# 게임 규칙 명세서 (구현 기준)

- 문서 버전: v1.0
- 작성일: 2026-03-04
- 기준: 현재 프론트엔드 구현 코드(`src/components/game/chess.jsx`, `src/components/game/janggi.jsx`, `src/game/rules/*`)

---

## 1. 목적

이 문서는 **현재 코드에 실제로 구현된 규칙**을 기준으로, 체스/장기/오목 혼합 대전의 진행 방식과 판정 로직을 정의한다. 

- 맵별 규칙 차이(체스판/장기판)
- 진영(체스/장기/오목)별 행동 가능 규칙
- 턴, 타이머, 승패 판정, 준비/시작 조건
- 예외/제한 및 현재 미사용 규칙

> 참고: 일반적인 정통 체스/장기 규칙과 일부 다를 수 있으며, 본 명세는 “게임 디자인 의도”가 아닌 “현재 구현 동작”을 우선한다.

---

## 2. 공통 개념

### 2.1 좌표/진영/용어

- `side`: `top` 또는 `bottom`
- `faction`: `chess`, `janggi`, `omok`
- 보드 셀은 `{row, col}`로 표현
- 렌더링 시 내 관점에서 보드가 뒤집혀 보여도, 내부 판정은 보드 좌표 기준으로 처리

### 2.2 맵 타입

- `체스판` → `Chess` 컴포넌트 사용 (8x8)
- `장기판` → `Janggi` 컴포넌트 사용 (9x10)

### 2.3 게임 시작 전 준비 단계

- 각 플레이어는 배치 모드/배치를 확정하고 `준비` 상태로 전환
- 호스트만 `시작` 가능
- 봇 방은 봇 측 준비를 자동 완료 처리

### 2.4 턴/타이머

- 턴 제한시간은 방 설정 `turnSeconds`를 사용 (정규화 범위: 1~600초, 기본 60초)
- 카운트다운은 클라이언트에서 진행
- `0초` 도달 시 **호스트가 턴 전환 패치**를 전송
- 경고음은 기본 5초 이하에서 초당 1회 트리거

### 2.5 공통 승리 개념

현재 구현상 맵 공통으로 자주 쓰이는 종결 트리거:

1. 상대 왕/궁(`king`) 포획
2. 오목 `5목` 완성(오목 진영 턴)
3. (체스판 전용) 비오목 진영의 오목 돌 제거 누적 목표 달성

> `winnerReason`에 `checkmate`, `timeout` 라벨은 포맷 존재하나, 현재 코드 경로에서는 실질적으로 `capture`, `connect5` 중심으로 설정된다.

## 3. 맵 A: 체스판(8x8) 규칙

## 3.1 시작 배치

### 3.1.1 체스 진영

- 기본 체스 16기물(룩/나이트/비숍/퀸/킹/비숍/나이트/룩 + 폰 8)
- 체스 진영은 커스텀 배치를 사용할 수 없음(강제 기본 배치)

### 3.1.2 장기 진영

- 장기 포진 4종 선택 가능: `안상`, `안마`, `왼상`, `오른상`
- 필요 시 `custom` 배치 가능
- `custom` 모드에서는 내 화면 하단 3줄 영역에서만 배치 가능

### 3.1.3 오목 진영

- 시작 시 보드에 오목 돌 없음
- 오목 턴마다 빈 칸 1곳에 돌 배치

### 3.1.4 준비 가능 조건

- `custom` 모드인 경우 풀 기물을 모두 배치해야 준비 가능
- 그 외 모드는 즉시 준비 가능

## 3.2 턴 진행

1. 현재 턴의 진영이 `omok`이면 돌 배치 처리
2. 그 외(`chess` 또는 `janggi`)면 기물 선택 후 합법 이동 실행
3. 이동/착수 후 승패 검사
4. 미종료 시 턴 교대

## 3.3 오목 전용 상호작용(체스판 핵심)

체스판에서는 오목 돌이 단순 장식이 아니라 전장 오브젝트로 동작한다.

### 3.3.1 돌 배치

- 오목 진영은 빈 칸에만 착수 가능

### 3.3.2 돌 포획/파괴

- 비오목 기물은 오목 돌 칸으로 이동해 돌을 제거 가능
- 돌을 제거한 기물은 `frozen=true` 상태가 되어 **다음 자기 턴에 이동 불가**
- 해당 진영의 다음 이동 직전에 동결이 해제되는 구조

### 3.3.3 포위 제거(질식)

- 오목 돌이 어떤 비오목 기물을 상/하/좌/우 4방향에서 모두 인접하게 막으면 그 기물 제거
- 제거된 기물은 오목 착수자의 포획 목록에 누적

### 3.3.4 오목 승리

- 오목 돌이 가로/세로/대각선 중 하나로 5개 이상 연속이면 즉시 승리

### 3.3.5 비오목 진영의 돌 제거 승리

- 비오목 진영은 누적 제거한 오목 돌 수가 `omokStoneTarget` 이상이면 승리
- 목표값 정규화 범위: 6~12 (기본 8)

## 3.4 체스 기물 이동 규칙(체스판)

- 폰: 전진 1칸, 첫 이동 2칸, 대각선 공격
- 앙파상(en passant): 조건 만족 시 허용
- 룩/비숍/퀸: 슬라이딩 이동
- 나이트: 점프 이동
- 킹: 8방향 1칸
- 캐슬링: 룩/킹 미이동 + 경로 비어 있음 조건에서 허용
- 폰 프로모션: 상대 끝 랭크 도달 시 자동 `queen` 승격

## 3.5 장기 기물 이동 규칙(체스판에서 장기 진영일 때)

- 병: 전진 + 좌우 이동
- 차: 직선 슬라이딩
- 포: 스크린(중간 기물) 1개 필요, 포는 포를 잡을 수 없음
- 마/상: 길목 차단 규칙 반영
- 궁/사: 궁성(3x3) 내부 이동

> 체스판에서 장기 궁성은 내부 규칙 함수 기준으로 좌표 영역에 맞춰 제한된다.

## 3.6 현재 구현의 체크/체크메이트 관련 상태

- 내부 유틸에 `isKingInCheck`, `hasAnyEscapeMove`가 존재
- 단, 현재 메인 전투 루프에서는 체크/체크메이트로 게임 종료시키는 흐름이 기본 승패 경로가 아님
- 실질 종료는 `왕 포획`, `5목`, `돌 제거 목표 달성` 중심

## 4. 맵 B: 장기판(9x10) 규칙

## 4.1 시작 배치

### 4.1.1 장기 진영

- 고정 배치 사용(왕 1, 사 2, 차 2, 마 2, 상 2, 포 2, 졸 5)

### 4.1.2 체스 진영

- 추천 배치(`left`/`right`) 또는 자율배치(`custom`) 선택 가능
- `custom`은 자기 진영 3줄 영역에서만 허용

### 4.1.3 오목 진영

- 시작 기물 없음, 턴마다 빈 칸 1곳에 착수

### 4.1.4 준비 가능 조건

- 체스 진영이 `custom`이면 체스 풀 기물 전부 배치해야 준비 가능
- 그 외는 준비 즉시 가능

## 4.2 턴 진행

- 턴 주체의 진영이 오목이면 돌 착수
- 아니면 선택한 기물의 합법 이동 실행
- 행동 후 승리 여부 검사, 미종료 시 턴 교대

## 4.3 오목 규칙(장기판)

- 오목 진영은 빈 칸에만 착수
- 5목 완성 시 즉시 승리
- 체스판과 달리 장기판 구현 루프에는 다음이 없음:
  - 오목 질식(4방향 포위 제거)
  - 돌 제거 수 목표 승리
  - 돌 포획 후 동결 처리

## 4.4 기물 이동 규칙(장기판)

장기판 룰 엔진은 진영별 이동을 아래처럼 처리한다.

### 4.4.1 체스 기물

- 폰: 전진 1칸 + 대각선 공격
- 룩/비숍/퀸/나이트/킹: 기본 이동 구현
- 체스판 전용 특수 규칙(캐슬링/앙파상/프로모션)은 장기판 구현에서 사용되지 않음

### 4.4.2 장기 기물

- 병/차/포/마/상/궁/사 이동 규칙 반영
- 궁성 내부 대각 이동 조건 포함
- 포는 스크린 규칙 사용, 포-포 직접 포획 불가

## 4.5 승리 조건(장기판)

- 상대 왕(`king`) 포획 시 즉시 승리
- 오목 진영은 5목으로 승리 가능
- 시간 만료는 즉시 패배가 아니라 턴 강제 전환(호스트 패치)

---

## 5. 봇 대전 규칙

## 5.1 공통

- 봇 ID 접두사(`[BOT`)로 봇 플레이어 판별
- 봇 턴 딜레이: 약 360~760ms 랜덤
- 난이도:
  - Lv1: 랜덤 선택
  - Lv2: 휴리스틱 선택
  - Lv3: Worker(`src/workers/bot-worker.js`) 기반 탐색  

## 5.2 오목 턴 봇

- 빈 칸 후보 중 랜덤 또는 휴리스틱 점수 최대 착수
- 5목 가능 수에 큰 가중치

## 5.3 기물 턴 봇

- 합법 이동 중 점수 최대 이동 선택(중앙성/포획 가치/위험도 등)
- 선택 가능한 수가 없으면 턴 넘김 처리

---

## 6. 설정값 명세

- 턴 제한시간: `room.turnSeconds` (1~600, 기본 60)
- 오목 돌 제거 승리 목표(체스판): `room.omokStoneTarget` (6~12, 기본 8)
- 봇 난이도: `room.botLevel` (1~3)

---

## 7. 상태 필드(`gameSetup`) 핵심

공통적으로 사용되는 주요 필드:

- `started`: 대전 시작 여부
- `firstTurn`, `turnSide`: 선공/현재 턴
- `winner`, `winnerReason`: 승자/승리 사유
- `battleBoard`: 전투 보드 상태
- `lastMove`: 직전 수(특수 규칙 판정에 사용)
- `capturedBySide`: 진영별 포획 목록
- 준비/배치 관련: `p1Ready`, `p2Ready`, `p1Mode`, `p2Mode`, `*CustomLayout`, `*Formation` 등

---

## 8. 승리 사유 코드

- `capture`: 왕/궁 포획 또는 (체스판에서) 돌 제거 목표 달성
- `connect5`: 오목 5목 완성
- `checkmate`: 라벨은 존재하나 현재 주요 종료 경로에서 사용 빈도 낮음
- `timeout`: 라벨은 존재하나 현재 구현은 시간초과 즉시 패배 대신 턴 교대

---

## 9. 구현 기준 주의사항

1. 이 명세는 현재 프론트 구현 동작 기준이다.
2. 서버가 별도 강제 검증을 하지 않는 경우, 클라이언트 패치 로직이 사실상 규칙 원본이 된다.
3. 향후 체크/체크메이트/시간패를 정식 도입하려면 종료 판정 우선순위를 재정의해야 한다.

---

## 10. 향후 확장 제안(옵션)

- 체크메이트/스테일메이트를 실제 종료 규칙으로 승격
- 시간 만료 시 턴 넘김/즉시 패배를 룸 옵션화
- 장기판에도 체스판형 오목 상호작용(질식/동결/돌 제거 승리) 적용 여부를 설정화
- 룰 버전(`ruleVersion`) 필드 도입으로 서버/클라 규칙 동기화 강화
